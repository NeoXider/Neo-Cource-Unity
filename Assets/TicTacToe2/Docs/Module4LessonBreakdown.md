# Разбор по урокам модуля 4

Этот файл нужен, чтобы готовая игра легко раскладывалась на уроки `M4У1`-`M4У5`.

## С чего стоит начать

Самый правильный старт для детей:

1. Сначала собрать поле `3 x 3` из кнопок.
2. Сделать только два скрипта:
   - `BoardCell.cs`
   - `TicTacToeGame.cs`
3. На первом уроке не пытаться сразу сделать всю игру.
4. В конце первого урока дети обязательно должны увидеть результат:
   - на кнопки можно нажимать;
   - в клетке появляется символ.

То есть первый урок не про "готовые крестики-нолики", а про понятный старт:

- есть поле;
- есть массив;
- есть связь кнопки и кода;
- есть видимый результат на экране.

## Общая стратегия на 4 урока

Нужно уложить основную игру в `4` занятия, а `5`-й оставить только на доработку.

Поэтому лучший ритм такой:

### Урок 1

- поле;
- кнопки;
- массив;
- клик по клетке;
- все клетки пока ставят один и тот же символ.

### Урок 2

- очередность `X` / `O`;
- занятая клетка не нажимается второй раз;
- победа;
- ничья.

### Урок 3

- `List<BoardCell>`;
- `foreach`;
- очистка поля;
- блокировка и разблокировка клеток;
- нормальный режим игры для двух игроков;
- подготовка структуры под бота.

### Урок 4

- `Random.Range`;
- простой бот;
- выбор режима: против бота или вдвоём;
- базовая полировка.

### Урок 5

- доводка;
- тесты;
- исправление багов;
- сборка и публикация.

## M4У1 - поле, кнопки, массив, первый клик

### Главная цель урока

Собрать игровое поле и дать детям быстрый визуальный результат.

### Какие темы проходят

- массивы
- индекс элемента массива
- связь UI-кнопки и кода
- первый клик по клетке

### Что нового узнают

- что массив нужен, когда надо хранить много однотипных значений;
- что у каждого элемента массива есть индекс;
- что поле `3 x 3` удобно хранить в массиве из `9` элементов;
- что кнопка может передавать в игру номер клетки.

### Что будут использовать на практике

- `int[] board = new int[9]`
- индекс клетки `cellIndex`
- кнопку `Button`
- текст `TMP_Text`
- вызов метода из одной клетки в главный скрипт игры

### Какие файлы создаются

- `Assets/TicTacToe2/Scenes/TicTacToe.unity`
- `Assets/TicTacToe2/Scripts/TicTacToeGame.cs`
- `Assets/TicTacToe2/Scripts/BoardCell.cs`

### Что делаем на сцене

- создаём `Canvas`
- создаём `GamePanel`
- создаём 9 кнопок `Cell0` - `Cell8`
- создаём объект `GameManager`

### Что делаем в коде

#### `BoardCell.cs`

Дети делают:

- `cellIndex`
- ссылку на кнопку
- ссылку на текст
- вызов `game.HandleCellClicked(cellIndex)`

Какие поля пишут:

- `public int cellIndex`
- `public Button button`
- `public TMP_Text symbolText`
- `public TicTacToeGame game`

Какие методы пишут:

- `Awake()`
- `Setup(TicTacToeGame ticTacToeGame, int newCellIndex)`
- `OnCellClicked()`
- `SetSymbol(string symbol)`

#### `TicTacToeGame.cs`

Дети делают:

- `int[] board = new int[9]`
- обработку индекса клетки
- запись значения в массив
- первый показ символа в клетке

Какие поля пишут:

- `public int[] board = new int[9]`
- `public string xSymbol = "X"`
- `public BoardCell[] cells` или `public BoardCell firstCell` как самый первый простой вариант

На первом уроке лучше действительно **не вводить `List<BoardCell>`**.
Пусть стартовая версия будет максимально простой:

- есть массив `board`;
- есть клетки;
- каждая клетка знает свой индекс;
- `TicTacToeGame` только получает индекс и меняет текст.

Если хочется совсем мягкий старт, можно:

- сначала сделать работу даже с одной кнопкой;
- потом быстро расширить до 9 кнопок;
- или сразу сделать `BoardCell[] cells`, но без `List`.

Какие методы пишут:

- `Start()`
- `HandleCellClicked(int cellIndex)`
- `UpdateCell(int cellIndex)`

`PrepareBoardCells()` на первом уроке можно не вводить вообще.
Его удобно добавить позже, когда появится нормальная коллекция клеток.

### Что важно объяснить

- массив хранит состояние поля;
- у каждой кнопки есть свой индекс;
- кнопка сообщает в игру только номер клетки;
- пока можно упростить и ставить один и тот же символ, например `X`.

### Что должно работать в конце урока

- есть поле `3 x 3`;
- по кнопкам можно нажимать;
- в клетке появляется символ;
- дети видят связь между кнопкой, индексом и массивом.

### Что не надо делать на этом уроке

- не делать сразу очередь хода;
- не делать победу;
- не делать бота;
- не делать режимы игры;
- не перегружать урок UI-логикой.

## M4У2 - очередь хода, победа, ничья

### Главная цель урока

Сделать уже настоящую игру на двух игроков.

### Какие темы проходят

- массивы в игровой логике
- состояния клетки
- условные операторы
- проверка победы и ничьей

### Что нового узнают

- что массив может хранить не просто данные, а состояние игры;
- как кодировать состояние клетки числами:
  - `0` - пусто
  - `1` - X
  - `2` - O
- как после каждого хода проверять правила игры;
- как по очереди переключать игроков.

### Что будут использовать на практике

- `currentPlayer`
- `if`
- `SwitchPlayer()`
- `CheckWinner(...)`
- `MatchesLine(...)`
- `IsBoardFull()`
- `FinishGameWithWinner(...)`
- `FinishGameWithDraw()`

### Какие файлы продолжаем менять

- `Assets/TicTacToe2/Scripts/TicTacToeGame.cs`
- `Assets/TicTacToe2/Scripts/BoardCell.cs`

### Что добавляем

- `currentPlayer`
- `1` = X
- `2` = O
- проверку "клетка уже занята"
- переключение хода
- проверку победы
- проверку ничьей

#### `TicTacToeGame.cs`

Какие поля добавляются:

- `public int currentPlayer = 1`
- `public string oSymbol = "O"`
- `public bool isGameOver`

Какие методы добавляются или расширяются:

- `HandleCellClicked(int cellIndex)`
- `MakeMove(int cellIndex)`
- `SwitchPlayer()`
- `CheckWinner(int player)`
- `MatchesLine(int player, int a, int b, int c)`
- `IsBoardFull()`
- `FinishGameWithWinner(int winner)`
- `FinishGameWithDraw()`

#### `BoardCell.cs`

Что меняется:

- `SetSymbol(string symbol)` теперь получает уже нужный символ (`X` или `O`) из `TicTacToeGame`
- клетка после хода может выключаться через `SetInteractable(bool isInteractable)`

Какие методы добавляются:

- `SetInteractable(bool isInteractable)`

### Что важно объяснить

- массив теперь хранит не просто "было нажатие", а состояние клетки;
- `0` - пусто, `1` - X, `2` - O;
- после каждого хода нужно проверить:
  - победил ли игрок;
  - заполнено ли всё поле.

### Что должно работать в конце урока

- X и O ходят по очереди;
- повторно нажать занятую клетку нельзя;
- игра умеет определять победу;
- игра умеет определять ничью.

## M4У3 - List, foreach, удобная работа с полем

### Главная цель урока

Показать, зачем кроме массива нужен ещё и список визуальных клеток.

### Какие темы проходят

- `List`
- `foreach`
- работа со списком объектов
- подготовка структуры под режимы игры

### Что нового узнают

- чем `List` отличается от массива;
- почему удобно хранить визуальные объекты отдельно от логики массива;
- как пройтись по всем клеткам без копирования одинакового кода;
- зачем нужен `foreach`, когда мы работаем с набором объектов.

### Что будут использовать на практике

- `List<BoardCell>`
- `foreach`
- `PrepareBoardCells()`
- `ResetBoard()`
- `SetBoardInteractable(...)`
- `UpdateStatusText()`
- `ShowModeSelection()`
- `StartNewMatch()`

### Какие файлы меняем

- `Assets/TicTacToe2/Scripts/TicTacToeGame.cs`
- `Assets/TicTacToe2/Scripts/BoardCell.cs`
- `Assets/TicTacToe2/Scripts/TicTacToeUI.cs`

### Что вводим

- `List<BoardCell> boardCells`
- цикл `foreach`
- текст статуса
- очистку поля
- включение и выключение клеток

#### `TicTacToeGame.cs`

Какие поля здесь уже должны быть:

- `public List<BoardCell> boardCells`
- `public bool playWithBot`
- `public bool gameStarted`
- `public bool isGameOver`
- `public TicTacToeUI ui`

Какие методы удобно писать на этом уроке:

- `PrepareBoardCells()`
- `ShowModeSelection()`
- `StartNewMatch()`
- `ResetBoard()`
- `SetBoardInteractable(bool isInteractable)`
- `UpdateStatusText()`

Именно на этом уроке удобно сделать переход от простой стартовой версии к более удобной:

- раньше клетки могли быть подключены "в лоб";
- теперь появляется `List<BoardCell>`;
- теперь можно централизованно подготовить все клетки через `PrepareBoardCells()`.

Где именно используется `foreach`:

- в `ResetBoard()` для очистки всех клеток;
- в `SetBoardInteractable(...)` для включения и выключения всех кнопок;
- в `UpdateCell(...)` для поиска нужной клетки по индексу.

#### `BoardCell.cs`

Какие методы тут уже используются:

- `ClearCell()`
- `SetInteractable(bool isInteractable)`

#### `TicTacToeUI.cs`

На этом уроке лучше ввести отдельный простой UI-скрипт.

Какие поля пишут:

- `public TMP_Text statusText`
- `public GameObject modePanel`
- `public GameObject gamePanel`
- `public TicTacToeGame game`

Какие методы пишут:

- `Bind(TicTacToeGame ticTacToeGame)`
- `ShowModeSelection()`
- `ShowGamePanel()`
- `SetStatusText(string text)`

### Где удобно использовать foreach

- пройтись по всем клеткам и очистить их;
- пройтись по всем клеткам и включить/выключить кнопки;
- пройтись по всем клеткам и подготовить поле к новой игре.

### Почему это хороший урок для foreach

Потому что дети сразу видят практическую пользу:

- не нужно писать 9 одинаковых строк;
- можно обработать все клетки одним циклом;
- `foreach` здесь понятнее, чем на абстрактных числах.

### Что должно работать в конце урока

- полноценная игра для двух игроков;
- есть текст "Ход игрока X" / "Ход игрока O";
- после конца игры поле блокируется;
- поле можно легко подготовить к новой партии;
- код стал чище и удобнее.

### Что ещё можно подготовить на этом уроке

- `bool playWithBot`
- `bool gameStarted`
- `bool isGameOver`

Это пригодится для следующего урока, где будет бот.

## M4У4 - Random.Range и простой бот

### Главная цель урока

Познакомить детей с `Random.Range` и сделать самого простого бота.

### Какие темы проходят

- `Random.Range`
- случайный выбор элемента
- поиск свободных клеток
- режим игры с ботом

### Что нового узнают

- как получать случайное число в Unity;
- как случайность используется в играх;
- как сначала собрать список подходящих вариантов, а потом выбрать один случайный;
- как подключить простого бота в уже готовую игру.

### Что будут использовать на практике

- `Random.Range(0, freeCells.Count)`
- `ChooseMove(int[] board)`
- `GetFreeCells(int[] board)`
- `GetRandomMove(List<int> freeCells)`
- `StartBotGame()`
- `MakeBotMove()`
- `OnBotClicked()`

### Какие файлы меняем

- `Assets/TicTacToe2/Scripts/TicTacToeBot.cs`
- `Assets/TicTacToe2/Scripts/TicTacToeGame.cs`
- `Assets/TicTacToe2/Scripts/TicTacToeUI.cs`
- `Assets/TicTacToe2/Scenes/TicTacToe.unity`

### Что добавляем

- кнопку режима "игра с ботом"
- `TicTacToeBot.cs`
- список свободных клеток
- `Random.Range`
- случайный ход бота

#### `TicTacToeBot.cs`

Это главный новый скрипт урока.

Какие поля можно не добавлять вообще:

- бот может обойтись без полей, только методами.

Какие методы пишут:

- `ChooseMove(int[] board)`
- `GetFreeCells(int[] board)`
- `GetRandomMove(List<int> freeCells)`

Что делает каждый метод:

- `ChooseMove(...)`:
  - получает массив поля;
  - собирает свободные клетки;
  - если ходить некуда, возвращает `-1`;
  - иначе возвращает случайную свободную клетку.
- `GetFreeCells(...)`:
  - проходит по массиву;
  - собирает индексы пустых клеток в список.
- `GetRandomMove(...)`:
  - использует `Random.Range(0, freeCells.Count)`;
  - возвращает случайный индекс из списка.

#### `TicTacToeGame.cs`

Какие поля добавляются:

- `public TicTacToeBot bot`
- `public bool playWithBot`

Какие методы добавляются или расширяются:

- `StartBotGame()`
- `StartTwoPlayersGame()`
- `MakeBotMove()`

Где вызывается бот:

- после хода игрока;
- если включён режим игры с ботом;
- если игра ещё не закончена.

#### `TicTacToeUI.cs`

Какие поля добавляются:

- `public Button twoPlayersButton`
- `public Button botButton`

Какие методы добавляются:

- `OnEnable()`
- `OnDisable()`
- `OnTwoPlayersClicked()`
- `OnBotClicked()`

Эти методы нужны, чтобы:

- подписаться на кнопки;
- переключать режим игры;
- запускать партию против бота или вдвоём.

### Какой бот нужен

Только самый простой:

- ищет все свободные клетки;
- выбирает одну случайно;
- ставит туда `O`.

### Почему это хороший уровень сложности

- дети успеют за урок;
- тема `Random.Range` естественно встраивается в игру;
- не нужно усложнять урок "умным" AI;
- остаётся время на проверку и отладку.

### Что должно работать в конце урока

- можно выбрать режим:
  - вдвоём
  - против бота
- после хода игрока бот делает случайный ход;
- бот ставит символ только в свободную клетку.
