# Настройки игры

Этот файл нужен как быстрый ориентир по значениям в инспекторе для `SpaceShooter2`.

Здесь не обязательно держаться каждого числа идеально.
Смысл файла в том, чтобы:

- быстрее собрать рабочую версию;
- не тратить урок на подбор случайных значений;
- иметь понятную стартовую точку для баланса.

## Что важно по стилю кода

Так как модуль 6 идёт после ООП, здесь уже полезно постепенно переходить на более аккуратный стиль:

- поля для инспектора часто делать как `[SerializeField] private`;
- `public` оставлять там, где нужен внешний доступ;
- общие данные игры менять через методы, а не напрямую;
- похожие сущности можно связывать через наследование.

То есть этот модуль подходит не только для механик, но и для первого более взрослого стиля Unity-кода.

## Player

Объект: `Player`

Скрипт: `PlayerShipController`

Рекомендуемые значения:

- `Move Speed` - `5`
- `Min X` - `-7`
- `Max X` - `7`
- `Min Y` - `-4`
- `Max Y` - `4`
- `Projectile Prefab` - префаб пули игрока
- `Fire Point` - дочерний объект перед кораблём
- `Game Manager` - ссылка на `GameManager` начиная с `Lesson3`

Что важно:

- если игрок слишком медленный, игра ощущается скучно;
- если слишком быстрый, управлять неудобно;
- `5` - хороший старт для первого теста.
- именно на этом скрипте удобно показать разницу между `public` и `[SerializeField] private`.

## Player Projectile

Префаб: `PlayerProjectile`

Скрипт: `ProjectileMover`

Рекомендуемые значения:

- если скорость слишком маленькая, выстрел не ощущается как шутер.

## Enemy

Префаб: `Enemy`

Скрипт: `EnemyShip`

Рекомендуемые значения:

- `Move Speed` - `2.5`
- `Destroy Y` - `-7`
- `Score Value` - `10`
- `Enemy Projectile Prefab` - префаб вражеской пули
- `Fire Point` - дочерний объект перед врагом
- `Start Delay` - `1`
- `Fire Rate` - `2`

- `2` секунды между выстрелами - хороший мягкий старт.

## Spawner

Объект: `Spawner`

Скрипт: `TimedSpawner`

Рекомендуемые значения:

- `Asteroid Prefab` - префаб астероида
- `Enemy Prefab` - префаб врага
- `Min X` - `-7`
- `Max X` - `7`
- `Spawn Y` - `6`
- `Asteroid Start Delay` - `1`
- `Asteroid Rate` - `1.5`
- `Enemy Start Delay` - `2`
- `Enemy Rate` - `3`
- `Game Manager` - ссылка на `GameManager` начиная с `Lesson3`

Что важно:

- астероиды должны появляться чаще врагов;
- врагов лучше делать реже, чтобы на сцене не было хаоса;
- если игра слишком сложная, сначала увеличивают интервалы, а не скорость всего подряд.
- здесь удобно использовать атрибуты Unity, чтобы красиво сгруппировать поля в инспекторе.

## GameManager

Объект: `GameManager`

Скрипт: `GameManager`

Рекомендуемые значения:

- `Score` - `0`
- `Is Game Over` - `false`
- `Speed Multiplier` - `1`
- `Speed Step` - `0.2`
- `Difficulty Rate` - `10`

Что важно:

- `Speed Multiplier` должен начинаться с `1`;
- `Speed Step` лучше делать маленьким, чтобы темп рос плавно;
- если игра слишком резко ускоряется, сначала уменьшают `Speed Step`.

## GameHud

Объект: `Canvas` или отдельный UI-объект

Скрипт: `GameHud`

Назначить ссылки:

- `Score Text` - текст очков
- `Status Text` - текст состояния игры
- `Game Over Panel` - панель конца игры
- `Game Manager` - ссылка на `GameManager`

Стартовое состояние:

- `ScoreText` показывает `Score: 0`
- `StatusText` показывает, что игра идёт
- `GameOverPanel` выключена

## Настройка событий

Начиная с `Lesson3`, в `GameManager` используются события:

- `OnScoreChanged`
- `OnGameOver`

Они вызываются из кода, а `GameHud` подписывается на них через `AddListener`.

Что важно объяснить:

- это уже не встроенный `Button.onClick`, а свой игровой канал связи;
- очки и конец игры сообщаются в UI через событие;
- логика и UI становятся связаны аккуратнее.
- это хороший пример инкапсуляции: UI не меняет счёт сам, а только получает уведомление.

## Где удобно показать наследование

В шутере есть несколько похожих объектов:

- астероид
- враг
- вражеская пуля

У них часто повторяются части поведения:

- движение вниз;
- удаление за экраном;
- работа с общей скоростью игры.

Поэтому именно в модуле 6 удобно показать базовый класс и наследование от него.

Главное правило:

- наследование добавляем как упрощение уже похожего кода;
- не делаем его ради самой теории.

## Хорошая стартовая сложность

Если нужно быстро получить удобную первую рабочую версию, оставь такие значения:

- игрок `Move Speed = 5`
- пуля игрока `Speed = 10`
- астероид `Speed = 3`
- враг `Move Speed = 2.5`
- спавн астероидов каждые `1.5`
- спавн врагов каждые `3`
- рост сложности каждые `10` секунд

## Что крутить на 4 уроке

На уроке полировки лучше менять только такие вещи:

- скорость игрока
- скорость пуль
- частоту спавна
- частоту выстрела врага
- тексты UI
- звуки
- экран `Game Over`

Лучше не добавлять в этот момент новые большие системы.
